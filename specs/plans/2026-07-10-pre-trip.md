## PR A — trip mode: today-band + offline stay card

**`lib/trip.js`** (pure, node-tested):
- `isStay(e)` = `/stay:/i` on the title — the baked data's convention (🏠/🛏️/🏔️ Stay: …).
- `stayForNight(events, iso)` — the stay whose `[date, endDate)` covers the NIGHT of iso
  (endDate = checkout morning, exclusive). Ties (shouldn't happen): prefer booked.
- `stayBooked(e)` = NOT `/not booked/i` in the title.
- `tripWindow(events, iso)` — chain stays where `next.date ≤ prev.endDate` (contiguous or
  overlapping) into a window `{start, end, day, total, stays}`; `end` = last checkout day
  (the fly-home day, INCLUDED — day 10/10 with no stay that night); `day` = 1-based.
  Returns null when iso outside every chain. User-created stay events participate when the
  caller passes merged events (dashboard does; emergency renders baked-only — documented).

**Dashboard band** (in `renderToday`, above the weather strip): when `tripWindow` is active —
- normal: `✈️ Trip day 3/10 · tonight: K's House Hokkaido (Asahidake Onsen)` → links `#/emergency`
- tonight unbooked: gold/alarm variant `⚠ tonight: NOT BOOKED — book by Jul 12` → links `#/emergency`
- last day (no stay tonight, iso === end): `✈️ Trip day 10/10 — heading home`
One line, esc()'d, no animation (glanced daily).

**Emergency stay section** (`emergency.js`, before the numbers section, only while the
window is active): "🛏 Tonight's stay" — tonight's card (title, area, date range,
bookingNotes, source link, BOOKED badge or NOT-BOOKED + book-by warning) + a compact list
of the remaining stays with their status. Pure render from `data.calendar`; offline-safe by
construction. Read-only page stays read-only.

CSS: `.w-tripband` (+ `.w-tripband-warn`), `.em-stay*`. SW → jwh-v293.

## PR B — quick-spend dashboard widget

- `index.html`: `<div class="widget" id="wSpend">` after `#wReadiness` (`data-i18n
  head.widget.spend`, STRINGS entry added — the frame-parity convention).
- `dashboard.js renderSpend()`: one-line form (`placeholder="1200 ramen"`, enterkeyhint,
  inputmode text), today + this-month totals under it (`monthTotal` from lib/spend.js).
  Submit → `parseSpend(value, nowISO())` → prepend `{id:'s'+Date.now(), …}` →
  `set(KEYS.spend, {v:1, items: pruneSpend(...)})` → dispatch `jwh:data-changed` → hint on
  parse failure (same copy as the budget page), refocus input, clear value. The budget page
  reads the same key; the dashboard refresh listener re-renders the widget.
- Widget participates in the existing widget-order drag (generic `.widget` handling).
- SW → jwh-v294.

**Verify:** CDP — type "1200 ramen" + Enter → totals update + jwh-spend-v1 contains the
item + budget page actuals show it; parse-failure hint; 44px touch height; 0 exceptions.
